### 2. Action Type Mismatch in Confirm Route
**File:** `src/app/api/chat/confirm/route.ts` line 87
- Code has `case 'create_checklist'` but `executeTool` returns `action_type: 'create_checklist_instance'`
- **Impact:** Checklist creation confirmations will fail with "Unknown action type"

### 3. Unhandled Tool Execution Errors in SSE Stream
**File:** `src/app/api/chat/stream/route.ts` lines 220-241
- If `executeTool()` throws inside `Promise.all()`, the entire stream dies without sending `tool_end` events
- Frontend sees an abrupt disconnection with no error message
- **Fix:** Wrap each executeTool call in try/catch within the Promise.all map

## HIGH SEVERITY ISSUES

### 4. No Role-Based Access Check in executeTool
**File:** `src/lib/chat/shared.ts` line ~408
- Tools are filtered at the model level (only role-allowed tools sent to Claude), but `executeTool()` itself doesn't validate roles
- If Claude hallucinates a tool name or if a tool is called directly, it executes without permission check
- **Fix:** Add `allowedRoles` check at the top of executeTool

### 5. Partial Message Loss on Stream Error
**File:** `src/app/api/chat/stream/route.ts` lines 289-296
- On error, saves an error message to DB but discards any `fullText` already streamed to the user
- User sees partial response in the stream but the saved message is only the error
- **Fix:** Save `fullText` (if any) before/alongside the error message

### 6. Missing Task Status in TypeScript Types
**File:** `src/lib/types.ts` line ~45
- TypeScript: `status: 'todo' | 'in_progress' | 'done'`
- Database: `check (status in ('todo', 'in_progress', 'review', 'done'))`
- Missing `'review'` status will cause type errors for tasks in review state

## MEDIUM SEVERITY ISSUES

### 7. Model Router Regex Too Broad
**File:** `src/lib/chat/model-router.ts` line 5
- `assess.*rating` matches any message with "assess" and "rating" anywhere — even "How do I assess room rating?"
- `exceeding` triggers Opus even for simple NQS mentions
- **Fix:** Tighten to `assess\w*\s+.*?rating` and `exceeding\s+(standards|indicators|rating)`

### 8. Missing useEffect Dependency in ChatAssistant
**File:** `src/components/ChatAssistant.tsx` line ~58
- useEffect checks `streamingMessage.text` but dependency array only has `streamingMessage?.isStreaming` and `streamingMessage?.messageId`
- Could miss finalizing a message if text changes without those values changing
- **Fix:** Add `streamingMessage?.text` to the dependency array

### 9. Duplicate ROLE_LABELS Definition
**Files:** `src/lib/types.ts` line ~1056, `src/lib/chat/shared.ts` line ~10
- Same constant defined in two places — maintenance risk
- **Fix:** Import from `types.ts` in shared.ts, or consolidate to one location

### 10. buildSystemPromptCached Not in Shared Module
**File:** `src/app/api/chat/stream/route.ts` lines 318-346
- Cache-aware prompt builder lives only in the stream route, not in shared.ts
- Fallback route has its own caching approach that could diverge
- **Fix:** Move to shared.ts

### 11. Prompt Caching Comment Inaccuracy
**File:** `src/app/api/chat/stream/route.ts` lines 343-344
- Comment says "cached for 5min" but Anthropic's `ephemeral` cache is request-scoped
- Ephemeral cache *does* persist for ~5 minutes on Anthropic's servers between requests with the same prefix
- The implementation is actually correct but the split strategy could be improved (static instructions as block 1, dynamic context as block 2)

### 12. RLS Policies Too Permissive on LMS and SharePoint Tables
**Files:** `supabase-migration-lms.sql`, `supabase-migration-sharepoint.sql`
- 18 tables use `USING (true) WITH CHECK (true)` — any authenticated user can read/write everything
- Includes: LMS modules, quiz responses, certificates, PDP reviews, SharePoint credentials
- **Fix:** Implement role-based RLS policies (admin/ns for write, all for read)
